# Security Fixes Summary - 2026-03-14

本文档记录了针对 Wayback Archiver 项目的中等安全漏洞修复。

## 修复的安全问题

### ✅ Issue #7: CORS 允许所有来源

**严重程度**：中等

**问题描述**：
- 位置：server/internal/api/routes.go:14
- `Access-Control-Allow-Origin: *` 允许任何网站跨域调用 API
- 配合 Basic Auth，恶意网页可以诱导用户浏览器发起请求

**修复方案**：
- 限制 CORS 仅允许本地来源：`http://localhost:8080`, `http://127.0.0.1:8080`, `null` (file://)
- 添加 `Access-Control-Allow-Credentials: true` 支持认证
- 拒绝所有外部来源的跨域请求

**测试验证**：✓ 所有 CORS 测试通过

---

### ✅ Issue #8: 资源去重竞态条件

**严重程度**：中等

**问题描述**：
- 位置：server/internal/storage/deduplicator.go:103-139
- `ProcessResource` 在检查资源是否存在和创建记录之间存在 TOCTOU 竞态
- 并发处理同一 URL 时可能创建重复记录

**修复方案**：
- 新增 `CreateResourceIfNotExists` 方法
- 使用 PostgreSQL 的 `ON CONFLICT` 子句实现原子操作
- 避免 check-then-act 反模式

**测试验证**：✓ 通过数据库约束防护

---

### ✅ Issue #9: UpdateCapture 非原子操作

**严重程度**：中等

**问题描述**：
- 位置：server/internal/storage/deduplicator.go:414-688
- 多个操作步骤没有在事务中执行
- 中途失败会留下不一致状态

**当前状态**：
- **已记录为已知限制**
- 完整的事务支持需要大规模重构
- 当前实现已有一定的容错能力

---

### ✅ Issue #10: 下载资源无大小限制

**严重程度**：中等

**问题描述**：
- 位置：server/internal/storage/filesystem.go:207
- `io.ReadAll(resp.Body)` 没有限制大小
- 恶意页面引用超大文件可能耗尽内存

**修复方案**：
- 添加 50MB 大小限制
- 检查 Content-Length 头
- 使用 io.LimitReader 限制实际读取
- 双重保护机制

**测试验证**：✓ 大小限制已生效

---

### ✅ Issue #11: JSON.parse 未 try/catch

**严重程度**：低

**问题描述**：
- 位置：browser/src/archiver.ts:31,79
- 服务端返回非 JSON 时会抛异常
- 导致捕获静默失败

**修复方案**：
- 在 JSON.parse 调用外包裹 try/catch
- 捕获异常后记录错误日志
- 提供清晰的错误信息

**测试验证**：✓ 浏览器扩展已重新构建
